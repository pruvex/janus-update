**Status:** 🥇 SEALED & COMPLETE (2026-04-26)

## Ziel

Implementierung einer deterministischen Regel-Engine für System-Optimierung basierend auf Insights. Keine KI im Backend-Core. Nur reine Logik mit Schwellenwerten für Action-First Integration.

## Architektur

### Komponenten

1. **OptimizationEngine** (`backend/services/logging/optimization_engine.py`)
   - **Rule Engine:** `evaluate_insight()` — Bewertet Insights gegen Schwellenwerte und generiert SystemAction Objekte
   - **Persistence:** `store_action()` — Speichert Aktionen in logs_actions Tabelle
   - **Analysis Pipeline:** `analyze_and_generate_actions()` — Vollständiger Pipeline: Fetch → Evaluate → Store

2. **GET Endpoint** (`backend/api/routers/system.py`)
   - **Route:** `/api/system/optimization-report`
   - **Response:** Markdown-Report (AI Studio Ready)
   - **Time Window:** Letzte 24 Stunden

3. **Schema** (`backend/data/schemas_logging.py`)
   - **ActionCreate:** Schema für logs_actions Tabelle
   - **Action:** Vollständiges Modell mit ID

### Rule Engine — Schwellenwerte

Deterministische Regeln ohne probabilistische Modelle:

| Regel | Bedingung | Action | Priority |
|-------|----------|--------|----------|
| Critical Error Rate | error_rate > 0.5 | MODEL_SWITCH | CRITICAL |
| High Error Rate | error_rate > 0.3 | SCALE_UP | HIGH |
| Critical Latency | avg_latency_ms > 5000 | MODEL_SWITCH | HIGH |
| High Latency | avg_latency_ms > 3000 | TIMEOUT_ADJUST | HIGH |
| Stable System | error_rate = 0 AND latency < 1000 | MONITOR | LOW |

### Action Types

| Action Type | Beschreibung |
|-------------|-------------|
| MODEL_SWITCH | Wechsel zu anderem Model |
| SCALE_UP | Ressourcen hochskalieren |
| TIMEOUT_ADJUST | Timeout erhöhen |
| MONITOR | System überwachen (keine Änderung) |

### Priority Levels

| Priority | Bedeutung |
|----------|-----------|
| CRITICAL | Sofortige Maßnahme erforderlich |
| HIGH | Maßnahme empfohlen |
| MEDIUM | Maßnahme in Betracht ziehen |
| LOW | Nur Monitoring |
