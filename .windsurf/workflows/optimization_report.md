# Port 8001: Janus Backend Production Port

# Skill: /optimization-report
**Description:** Liefert systemweite Optimierungsvorschläge basierend auf D12/D13.

**Ziel:** System-Health Monitoring mit rule-based Action-First Integration für AI Studio.

## Nutzung
Tippe in Windsurf: `/optimization-report`

Der Workflow ruft automatisch den Optimization-Endpunkt auf und zeigt das Ergebnis im Chat an.

## Ausführung

```bash
curl.exe -s http://localhost:8001/api/system/optimization-report
```

Optionale Parameter:
- `skill`: Filter report by specific skill name

**Beispiel mit Skill-Filter:**
```bash
curl.exe -s "http://localhost:8001/api/system/optimization-report?skill=knowledge.query"
```

## Rückgabe
Die Engine analysiert die letzten 24 Stunden Actions und gibt einen Markdown-Report zurück mit:
- **Generated:** Timestamp der Analyse
- **Actions:** Anzahl der generierten Aktionen
- **CRITICAL PRIORITY:** Sofortige Maßnahmen erforderlich
- **HIGH PRIORITY:** Maßnahmen empfohlen
- **MEDIUM PRIORITY:** Maßnahmen in Betracht ziehen
- **LOW PRIORITY:** Nur Monitoring

Für jede Action:
- **Skill / Model:** Skill und Model Name
- **Action:** Art der Maßnahme (MODEL_SWITCH, SCALE_UP, TIMEOUT_ADJUST, MONITOR)
- **Reason:** Grund für die Maßnahme
- **Current Value:** Aktueller Metrik-Wert
- **Threshold:** Schwellenwert, der die Maßnahme ausgelöst hat
- **Recommendation:** Konkrete Handlungsempfehlung
